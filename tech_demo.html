How this becomes reproducible

1
A seed is a starting number, fed into a pseudorandom number generator. seed = 18492731
2
The pseudorandom number generator uses that number to generate a sequence.
3
The Determinator uses that sequence to make its next decision, over and over.
plain terms

Same seed in, same sequence out, every time. That's why replaying it regenerates this exact bug instead of a similar one.

a pseudorandom number generator, precisely

A formula that takes a number in and spits a number out, and that output becomes the input for the next call. What it does with a number depends on the question being asked, but which number it gets never depends on the question.
